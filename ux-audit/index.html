Open Findings (8)

Active issues requiring attention. Includes carried items from v1 and new findings from v2 audit.

C4 Critical New in v2 Cybersecurity "Flexible Learning" label on pure VILT program

Cybersecurity Foundations is a pure VILT program (Online Live Classes only). But the progress breakdown on all 8 Cyber screens shows "Flexible Learning - 0%" alongside "Online Live Sessions - 0%". This is a cloning artifact from the TechMaster (Blended) template. It tells the learner they have self-paced content to complete when they don't.

All 8 Cybersecurity screens (header area). Check CE flow for same issue.

Remove "Flexible Learning" progress line from all Cybersecurity screens. Show only "Online Live Sessions - 0%".
C5 Critical New in v2 CE certificate thumbnail says "AI TechMaster Program"

The certificate preview thumbnail on CE/Certificates (569:3539) reads "AI TechMaster Program" instead of "Cloud Engineering Bootcamp". Cloning artifact not updated when CE flow was created from TM template.

CE / Certificates (569:3539)

Update certificate preview text to "Cloud Engineering Bootcamp". Check Cyber certificates for same issue.
M1 Major Empty/error/loading states still mostly undefined

Cyber/Recordings empty state is a good model. But empty states for Assessments, Projects, Certificates, Mentors, Course Q&A are still missing. No loading skeletons or error states defined anywhere. Developers will improvise without these.

Create 3 reference patterns using Cyber/Recordings as model: (1) empty list state, (2) loading skeleton, (3) error with retry.
M7 Major "Blocked" status pill still red in Filter Modal

Filter sidebar (523:2893) "Blocked" pill still has red background. While red for "blocked" could be semantic, it's the most visually aggressive element in the sidebar and inconsistent with the otherwise neutral filter UI.

Modal / Filter Sidebar (523:2893)

Change to neutral dark gray bg #646469 with white text.
M8 Major Q&A "UNANSWERED" could use neutral treatment

"ANSWERED" green is fine (success). "UNANSWERED" is currently gray which is acceptable. The visual weight between the two creates adequate distinction. Minor: consider brand blue for UNANSWERED to draw attention to questions needing answers.

DA / Course Q&A (557:766)

M9 Major New in v2 Recordings error card uses heavy red treatment

TM/Recordings shows a "Recording failed" card with full-width red/pink background, red X icon, red error text, and red "Report Issue" button. While red for errors is semantic, the combined treatment is the heaviest red usage in the entire wireframe set. Four different red elements on one card row. Also "1 session missing" text in red at top right.

TM / Recordings (523:3856) - Session 5 "Reinforcement Learning" card

Reduce red density: keep red text OR red bg, not both. "Report Issue" button could be outlined instead of red-filled. "1 session missing" could use dark text with a subtle icon instead of red text.
M10 Major New in v2 CE "Starting Soon" button appears green-tinted

On CE/Live Sessions (523:4289), the "Starting Soon" button appears to use a green-tinted fill rather than the standardized dark gray rgb(33,33,33) from Nelson's redesign. The TM version is correct. This suggests the CE flow wasn't fully updated.

CE / Live Sessions PD (523:4289)

Apply the same dark gray button treatment from TM/Live Sessions.
m7 Minor New in v2 DA tab "Flexible Learning" vs "Modules" naming

DA/PD first tab now reads "Flexible Learning" instead of "Modules". Aligns with delivery type taxonomy but is less descriptive. A learner looking for their module list may not intuit that "Flexible Learning" = their self-paced modules.

Verify this is intentional. If delivery-type tab names are the convention, it's consistent.
m8 Minor New in v2 DA affiliation banner "Course 3 of 5" should be "Course 2 of 4"

DA/PD (523:4070) shows "Part of AI TechMaster Program - Course 3 of 5". TechMaster has 4 courses (Python Basics, Data Analytics, Deep Learning, Capstone). Data Analytics is Course 2. Should read "Course 2 of 4".

Quick text fix.
M5 Open No explicit back navigation on course detail pages

Detail pages rely solely on breadcrumbs. Back arrow icon is small (12px). Acceptable for desktop, flag for mobile.

m3 Minor KPI stat cards lack visual hierarchy

5 KPI cards at top of every detail page look identical. Consider emphasizing the primary metric (progress % or Est. Completion).

m5 Minor Footer content is minimal

Could serve as secondary navigation for support, accessibility, language. Low priority.

  • Nelson's button redesign New session button system (black "Join Live" with green emoji, dark "Starting Soon", outlined "Watch/Review") is clear and consistent across TM flow.
  • CourseTypeBadge component system 3-variant component set with distinct colors per delivery type. Propagates cleanly across 81 instances.
  • SessionBanner (Option 1b) Slim notification banners inform without dominating. Good component variant architecture.
  • Program affiliation banner Clean conditional banner on DA/PD showing program context with "View Program" link.
  • Cyber Recordings empty state Good reusable model: icon + message + helpful context + related content. Should be replicated to other tabs.
  • Badge standardization 279+ badges at cornerRadius 100px with WCAG AA contrast. Structural work held up across both audits.
  • Grid compliance 3,086 values snapped to 4pt grid. 99% compliance. No regression from v1 to v2.
