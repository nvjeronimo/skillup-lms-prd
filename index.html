Usage Guidelines

Typography

Use DM Serif Display for headings only. Never for body text. DM Sans handles everything else: paragraphs, buttons, labels, navigation. Plus Jakarta Sans is reserved for small UI elements like badges, captions, and metadata.

The production platform uses proprietary Bureau Sans/Serif and Graphik. The fonts in this DS are open-source equivalents. Swap when you have the licensed files.

Color

Brand dark (#080C28) is not pure black. That specific RGB (8, 12, 40) gives dark surfaces an editorial quality. Use it for heroes, footers, and dark panels. Never use #000000.

Accent blue (#005FFF) is for interactive elements only: links, focus rings, primary actions on light backgrounds. Don't use it for decoration or large surfaces.

Category tint colors (purple, magenta, plum, crimson, burnt) are for badge backgrounds and feature card gradients. Each maps to a content category. Don't mix them.

Spacing

8px base grid. All spacing derives from it. Use the token names (--space-xs, --space-sm, etc.) not raw pixel values. Padding inside cards: 20-24px. Gaps between cards: 16-24px. Section spacing: 64-72px.

Components

Primary buttons are always dark (#0D0D0D). One per section. Don't stack two primary buttons side by side. Use secondary or ghost for the second action.

Cards never have rounded corners greater than 16px. Course listing cards use border-radius: 16px. Enrollment cards use 12px. Inputs use 10px. Keep the hierarchy consistent.

Elevation: shadow-sm for inputs and subtle elements. shadow-md for cards on hover and dropdowns. shadow-lg for modals and the enrollment sidebar only.

States

Every interactive element needs four states: default, hover, focus, disabled. Inputs also need error and success. Focus rings are always blue (#3E89FF), 2px solid, 2px offset.

Disabled elements use opacity: 0.4 with pointer-events: none. Don't change the color. Don't hide them. Show them grayed out so users understand what's available.

Motion

Keep it restrained. Hover transitions: 200ms ease. Scale/expand: 300ms ease-out. No page transitions. No scroll-triggered animations beyond the sidebar active state. The design language is editorial, not playful.
