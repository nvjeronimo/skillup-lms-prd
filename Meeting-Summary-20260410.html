VILT Architecture Discovery

Meeting summary and key decisions from the LMS walkthrough session

Date April 10, 2026
Duration 36 minutes
Participants Harpreet Kaur, Govind Sharma, Rupali Amit Mahale, Navdeep Malhotra, Nelson Jeronimo
Triggered by Rupali's request to show VILT topics alongside self-paced content in the LMS

Context

Rupali raised a concrete problem: TechMaster AIML has 16 self-paced courses with weekly VILT sessions, but learners can't see VILT topics inside the LMS course structure. The sessions, topics, and schedule only exist in emails from the Srishti team. She wanted VILT topics displayed as part of the course alongside self-paced modules.

Govind pushed back on the initial proposal. The team had previously burned themselves creating separate VILT and self-paced instances of the same course. Learners ended up going to two different places for the same content. This led to the core architectural principle that emerged from the meeting.

Core Principle

Resolved
Course structure and delivery method are separate concerns
VILT is a mode of delivery, not a course type. A course is a package of content on a specific topic. It has title, modules, lessons, quizzes, projects. Whether that content is delivered by an instructor, consumed self-paced, or both is a separate axis. Never create separate VILT and self-paced instances of the same course.
Govind Sharma
"The course has to be a course. The delivery is separate. VILT is a delivery. It has nothing to do with course creation."

Key Decisions

Resolved
Delivery mode tagging at lesson, module, or course level
Admin tags each lesson or module as self-paced, VILT, or blended when creating the course. If VILT is tagged, the system requires date, time, presenter, and meeting link. The tag is set on the admin side and reflected on the learner side.
Resolved
Delivery mode bubbles up automatically
If all lessons in a module are VILT, the module shows as VILT. If mixed, it shows as Blended. If all self-paced, it shows as Self-Paced. Same logic applies from module to course level. No manual tagging needed at parent levels.
Resolved
"Pure VILT" still includes quizzes, projects, and readings
Navdeep used the university analogy: you attend lectures, but you still have textbooks, assignments, and projects. A VILT course is heavily instructor-led, but it doesn't mean zero self-paced content. This distinction matters for UI: don't treat VILT as a stripped-down course.
Resolved
No prerequisite locking for VILT sessions
Learners should not be blocked from attending a VILT session even if they haven't completed prior modules. The system should nudge ("You haven't completed Module 1 and 2. Your session on Module 3 topics is in 3 days.") but never block. University model: the professor doesn't wait for you.
Resolved
Calendar at two levels: course tab + global view
Inside a course, a calendar tab shows only that course's VILT sessions with topics, date, time, presenter. At the global level (My Learning), a calendar aggregates all VILT sessions across all enrolled programs. Both views needed.
Open
Visual differentiation of delivery modes
The team agreed learners need to identify at a glance which modules/lessons are VILT vs self-paced. Options discussed: badge, icon, color coding. No specific solution chosen. Navdeep suggested VILT lessons should also show the scheduled date/time inline.

Impact on Product

Decisions Resolved

Design Implications
Course detail screens need delivery mode badges. Calendar component needed at two scopes. Nudge/reminder pattern for prerequisite gaps.

Next Steps

1 Define the visual language for delivery mode differentiation (badge vs icon vs color). Decide per-component: course card, module row, lesson row, calendar event.
2 Design the course-level calendar tab showing VILT sessions with topic, date, time, presenter, and meeting link.
3 Design the global calendar view aggregating all enrolled VILT sessions across programs.
4 Define the nudge/reminder UX for learners who haven't completed prerequisites before an upcoming VILT session.
5 Validate the bubble-up logic with edge cases: single VILT lesson in an otherwise self-paced module, recording-only sessions, rescheduled sessions.
